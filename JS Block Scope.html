JavaScript Let

-The let kewboard was introduced in ES6 (2015)
-Variable decleared with let have block scope.
-Variable decleared with let must be decleared before use.
-Variable decleared with letcannot be re-decleared in the same scope.

Block Scope

-Before Es6(2015),JavaScript did not have block scope.
-JavaScript had globle Scope and function scope.
-ES6 introduced the two new JavaScript keywords: let and const. These two keywords provided Block Scope in JavaScript:
Example: Variable declared inside a {}block cant be outside the block
{ let x =2; }
//x can NOT used here.

Global Scope

-Variable decleared with the var always have globle scope.
-Variable decleared with the var keyword can NOT have block scope:
Example:Variable decleared with var inside a {} block access can be accessed from outside the block:
{ var x =2; }
// x CAN used here.

Can not be Redecleared

Variables defined with let can not be decleared.
You can not accidentally redecleare a variable declear with let.
with let you can not do this:
let x = "John Doe";
let x = 0;

Redeclearing Variables

-Redeclearing a variables using the var keywords can impose problems.
-Redeclearing a variable inside a block will also redecleare the variable outside the block:
Example:
var x = 10;
// Here x is 10
{
var x = 2;
// Here x is 2
